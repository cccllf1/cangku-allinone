📝 API命名标准

⚠️ 重要：字段命名规范

本API严格遵循 snake_case 命名规范，禁止使用简化字段名：

  • ✅ 使用 product_code 而不是 code
  • ✅ 使用 product_name 而不是 name
  • ✅ 使用 sku_code 而不是 code
  • ✅ 使用 location_code 而不是 code
  • ✅ 使用 is_defective 而不是 defective
  • ✅ 使用 external_codes 而不是 external_code
🔍 通用字段说明

以下字段适用于所有修改操作（POST/PUT）：

字段名 类型 必需 说明
operator_id string 是 操作人ID，用于追踪谁进行了操作
is_urgent boolean 否 是否紧急操作，默认false
notes string 否 操作备注说明

📊 库存统计说明

此接口已整合库存统计功能，替代原 /inventory/by-product 接口：

  • product_total_quantity: 商品所有SKU的总库存
  • color_total_quantity: 每种颜色下所有尺寸的总库存
  • sku_total_quantity: 单个SKU在所有库位的总库存
  • locations: 每个SKU在各库位的详细库存分布
  • total_sku_count/total_location_count/total_color_count: 各层级的统计数量

category_name_2string否二级分类名称（如"上装"）

🎯 智能查询

此API会自动查询商品级和SKU级的外部条码，如果是SKU级外部条码，会返回 matched_sku 字段，前端可据此自动选择颜色尺码，提供与SKU编码查询相同的用户体验。

📥 成功响应示例:

{ "success": true, "data": { "product_id": "64f9a1b2c3d4e5f6789abc01", "product_code": "SHIRT-001", "product_name": "经典纯棉T恤", "unit": "件", "has_sku": true, "category_code_1": "CLOTHING", "category_name_1": "服装", "category_code_2": "TOPS", "category_name_2": "上装", "product_total_quantity": 150, "colors": [...], "description": "高品质纯棉面料制作", "external_code": { "code": "EXT123456", "source": "客户A", "description": "客户A的商品条码" }, "matched_sku": { "sku_code": "SHIRT-001-红-L", "sku_color": "红", "sku_size": "L", "stock_quantity": 25, "image_path": "/uploads/red-shirt.jpg" }, "created_at": "2024-01-01T10:00:00Z", "updated_at": "2024-01-15T14:30:00Z" }, "error_code": null, "error_message": null }

响应字段说明:

字段名类型说明
external_codeobject匹配的外部条码信息
matched_skuobject仅SKU级外部条码时返回，包含具体SKU信息
